The Agent Server API reference is available within each [deployment](/langsmith/deployments) at the `/docs` endpoint (e.g. `http://localhost:8124/docs`).

Browse the full API reference in the **Agent Server API** section in the sidebar, or see the endpoint groups below:

- [Assistants](/langsmith/assistants/create-assistant) - Configured instances of a graph
- [Threads](/langsmith/threads/create-thread) - Accumulated outputs of a group of runs
- [Thread Runs](/langsmith/thread-runs/create-run) - Invocations of a graph/assistant on a thread
- [Stateless Runs](/langsmith/stateless-runs/run-stateless) - Invocations with no state persistence
- [Crons](/langsmith/crons-plus-tier/create-cron) - Periodic runs on a schedule
- [Store](/langsmith/store/list-namespaces) - Persistent key-value store for long-term memory
- [A2A](/langsmith/a2a/get-agent-card) - Agent-to-Agent Protocol endpoints
- [MCP](/langsmith/mcp/list-tools) - Model Context Protocol endpoints
- [System](/langsmith/system/check-service-health) - Health checks and server info

## Authentication

For deployments to LangSmith, authentication is required. Pass the `X-Api-Key` header with each request to the Agent Server. The value of the header should be set to a valid LangSmith API key for the organization where the Agent Server is deployed.

Example `curl` command:

```shell
curl --request POST \
  --url http://localhost:8124/assistants/search \
  --header 'Content-Type: application/json' \
  --header 'X-Api-Key: LANGSMITH_API_KEY' \
  --data '{
  "metadata": {},
  "limit": 10,
  "offset": 0
}'
```
